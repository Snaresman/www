Store and query JSON documents

Shard and replicate a table in seconds with a simple API:

> r.table('users').configure(shards=2, replicas=3)

Apply replication settings to all tables in a database:

> r.db('my_app').configure(shards=2, replicas=3)

Set up server tags and scale to multiple datacenters:

> r.table('users').configure(shards=2, replicas={ 'us_east': 3, 'us_west': 2 }) Learn more →

Speed up queries with indexes

Query full cluster configuration details:

> r.db('system').table('table_config')

The `configure` command automatically picks optimal settings and writes them to the `table_config` system table. Update table configurations manually and place shards and replicas on specific servers for complete control: 

> r.db('system').table('table_config')
   .update(NEW_CONFIG)

Atomic updates

View detailed status of all tables in your cluster:

> r.db('system').table('table_status')

Or the status of all servers:

> r.db('system').table('server_status')

Subscribe to status changefeeds to monitor your cluster in realtime:

> r.db('system').table('server_status').changes() Learn more →

Distributed joins and subqueries

View disk, memory, and network statistics in your cluster:

> r.db('system').table('stats')

Use the query language to group data to quickly get the information you need:

> r.db('system').table('stats').group('server').avg('cpu')

Or query for specific statistics:

> r.db('system').table('stats').filter({'table': 'my_table'}) Learn more →

Geospatial queries

See disk compaction, cluster replication jobs, and long-running user queries:

> r.db('system').table('jobs')

Filter for specific jobs you're interested in:

> r.db('system').table('jobs').filter({'type': 'compaction'})

Kill long-running queries by deleting them from the `jobs` table:

> r.db('system').table('jobs').get(JOB_ID).delete() Learn more →

Binary data

Discover problems like unreachable servers quickly by seeing an actionable list of cluster issues:

> r.db('system').table('issues')

Query for specific issues to create specialized administration tools:

> r.db('system').table('issues').filter({'type': 'server_down'})
