Event-driven architecture

Polling is obsolete. Build realtime dashboards, leaderboards, and widgets with incremental updates to query results:

> r.table('games').max('score').limit(10).changes()

Create reactive experiences by subscribing to user events in realtime:

> r.table('games').filter({'user_id': 123}).changes()

Or get a firehose of changes from your database:

> r.table('games').changes() Learn more →

All the building blocks for great apps

Store, index, query, and modify JSON documents. Structure data your way with distributed joins. Build location-aware apps with geospatial queries and store multimedia with binary data support. Run analytics with Hadoop-style map/reduce.

Hassle-free operation at scale

Scale to multiple machines with a simple API:

r.table('games').configure(shards=5, replicas=3)

Precise control of your cluster when you need it:

r.table('games').table_status()
