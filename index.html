Built for realtime apps

Polling limits what you can build on the web. When your app polls for data, it becomes slow, unscalable, and cumbersome to maintain.

Instead, RethinkDB pushes data to your app in realtime via changefeeds:

r.table('games').filter({'user_id': 123}).changes()

Changefeeds change how you build realtime apps. Here's how you can build a realtime leaderboard, with incremental updates to query results:

r.table('games').max('score').limit(10).changes()

Or, just get a firehose of changes from your database:

r.table('games').changes()

Works with modern stacks

Get started by querying JSON documents with Python, Ruby, Node.js or dozens of other languages.

Build a modern app using your favorite framework, paired with realtime technologies like Socket.io or SignalR.

RethinkDB is packed with all the features you need to work with high velocity data, like distributed joins, geospatial queries, binary data, Hadoop-style map-reduce, and more.

Learn more go

Hassle-free operation at scale

When you're ready to scale your app, shard and replicate in a few clicks using an intuitive web UI.

If you need it, a simple API provides precise control over the cluster:

r.table('games').configure(shards=5, replicas=3)

Monitor your production cluster with live statistics and visibility into running jobs:

r.db('system').table('jobs').changes()
